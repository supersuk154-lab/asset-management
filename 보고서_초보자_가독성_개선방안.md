# 📖 보고서 초보자 가독성 개선 방안 (Easy Mode 리포트 설계안)

> 작성일: 2026-06-11
> 상태: **검토용 제안서 — 아직 구현하지 않음**
> 목적: 기존 마스터 리포트는 유지하되, 주식을 전혀 모르는 사람도
> **"아, 이런 이유 때문에 이렇게 투자하라는 거구나"** 를 30초~5분 안에 이해할 수 있는
> **별도의 쉬운 버전 리포트**를 추가하기 위한 설계 검토.

## 1. 문제 진단 — 초보자는 현재 리포트의 어디서 막히는가

실제 초보자 입장에서 기존 `master_report.md`를 읽을 때 막히는 지점을 짚어보면:

| # | 막히는 지점 | 구체 사례 (현재 리포트 기준) |
|---|------------|---------------------------|
| 1 | **전문용어 장벽** | "글라이드 패스(연령 굴절형)", "Core : Satellite", "리밸런싱 룰북", "IPS", "TAA" — 첫 화면부터 모르는 말이 연속으로 등장. HTML 툴팁이 있어도 '모르는 단어가 많다'는 인상 자체가 이탈 원인 |
| 2 | **정보 과부하** | 섹션이 10개 이상이고 전부 같은 무게로 나열됨. "꼭 알아야 할 것"과 "참고만 할 것"의 구분이 없음 |
| 3 | **숫자만 있고 번역이 없음** | "안전:위험 = 30:70" → 초보자는 *그래서 내 월급에서 얼마를 어디에 넣으라는 건지* 환산이 안 됨 |
| 4 | **인과관계가 흩어져 있음** | "당신 상황(원인) → 그래서 이 처방(결과)" 스토리가 섹션 2, 3, 5에 분산. 진단과 처방을 독자가 직접 연결해야 함 |
| 5 | **'읽는 보고서'지 '따라하는 보고서'가 아님** | TOP_ACTION이 한 줄 있지만, 나머지는 설명문. 초보자가 원하는 건 "오늘 뭘 누르면 되는지" |
| 6 | **두려움에 대한 답 부재** | 초보자의 1번 질문은 "떨어지면 어떡해요?"인데, 그 답(MDD, 3A 프레임워크)이 전문용어로 포장되어 있어 와닿지 않음 |

핵심 결론: **정보를 줄이는 게 아니라, "이해의 계단"을 만들어야 한다.**
전문가용 정보는 그대로 두고, 그 앞에 쉬운 층을 쌓는 방식이 정답.

## 2. 설계 원칙 5가지 (Easy Mode 헌법)

1. **용어 0 원칙**: 쉬운 버전에서는 전문용어를 '괄호로 설명'하는 게 아니라 **아예 쓰지 않는다.** 비유와 일상어로 완전 치환.
2. **모든 %는 원화로 번역**: "위험자산 70%" 금지 → "월 100만 원 중 70만 원" 으로만 표기.
3. **진단 → 이유 → 행동의 3박자 고정**: 모든 내용은 "당신은 지금 ___ 입니다(진단) → 왜냐하면 ___ (이유) → 그래서 ___ 하세요(행동)" 틀을 벗어나지 않는다.
4. **한 화면 = 한 메시지**: 섹션당 핵심 메시지 1개. 문장은 40자 이내, 문단은 3문장 이내.
5. **숫자는 기존 리포트와 1원도 다르지 않게**: 쉬운 버전도 reviewer의 서사적 환각 검사(V12) 대상. "쉽게" 쓰되 "다르게" 쓰면 안 됨.

## 3. 개선 아이디어 카탈로그 (효과 × 구현 난이도)

### A. 🥇 1페이지 Easy 요약 리포트 (강력 추천 — 핵심)
- 기존 리포트와 **별도 파일**(`{date}_easy.md` / `_easy.html`)로 생성.
- 구성: ① 한 문장 진단 → ② 왜 그런지 3가지 → ③ 이번 달 할 일 3가지 → ④ 내 돈 분배 그림 → ⑤ "떨어지면 어떡하죠?" → ⑥ 다음 점검일.
- 효과: ★★★★★ / 난이도: ★★☆ (템플릿 1개 + 치환 매핑 추가)

### B. 🥈 3레벨 점진적 공개 (Progressive Disclosure)
- Level 1: 30초 요약 (Easy 리포트 상단 카드)
- Level 2: 5분 설명 ("왜?"에 대한 Q&A)
- Level 3: 기존 마스터 리포트 (링크로 연결: "더 자세히 알고 싶다면")
- 효과: ★★★★☆ / 난이도: ★★☆

### C. 🥉 숫자 번역 5규칙 (Easy 버전 작문 규칙)
| 원래 표현 | 번역 규칙 | 예시 |
|----------|----------|------|
| 비율(%) | 월 여유자금 기준 원화 병기 | "70%" → "100만 원 중 70만 원" |
| 점수 | 신호등 + 생활 비유 | "73점 🟡" → "건강검진으로 치면 '재검 1~2개' 수준" |
| 큰 금액 | 월급 N달치로 환산 | "5,000만 원" → "월급 14달치" |
| 예상 손실 | 실제 내 돈 기준 금액 | "MDD -35%" → "최악의 해엔 70만 원 투자가 일시적으로 45만 원대로 보일 수 있어요" |
| 기간 | 구체적 날짜 | "90일 후" → "9월 9일" |
- 효과: ★★★★★ / 난이도: ★☆☆ (프롬프트 규칙만 추가)

### D. 비유 사전 (전문용어 → 일상 비유 고정 치환표)
| 전문용어 | Easy 버전 표현 |
|---------|---------------|
| 안전 버킷 / 위험 버킷 | **"지키는 돈" / "불리는 돈"** |
| Core / Satellite | **"밥 / 반찬"** (밥이 든든해야 반찬을 즐길 수 있다) |
| 글라이드 패스 | "나이에 맞는 자동 안전벨트" (젊을수록 느슨하게, 은퇴가 가까울수록 단단하게) |
| 리밸런싱 | "비중 원위치" (자동차 바퀴 정렬처럼 가끔 맞춰주는 것) |
| 분할 매수(DCA) | "월급날마다 자동으로 나눠 사기" |
| 비상예비비 | "에어백 통장" |
| 분산 투자 | "달걀 나눠 담기" (이미 익숙한 비유 활용) |
| 인적 자본 | "앞으로 벌 월급의 힘" |
| 꼬리위험 방어(금/달러) | "화재보험" (평소엔 심심하지만 큰일 날 때 빛남) |
- 효과: ★★★★☆ / 난이도: ★☆☆
- ⚠️ 주의: 비유는 **고정 치환표로 관리**해야 함. LLM이 매번 다른 비유를 창작하면 리포트마다 말이 달라져 신뢰가 깨짐 → compliance_rules.json에 `easy_mode_glossary` 블록으로 넣어 단일 진실 공급원화.

### E. "왜요?" Q&A 형식 (인과관계 노출)
- 처방마다 초보자가 실제로 물어볼 질문을 먼저 던지고 답하는 구조:
  - "Q. 왜 30만 원은 그냥 통장에 두라고 하나요?" → "A. 갑자기 병원비 같은 큰돈이 필요할 때, 투자한 걸 **손해 보면서 팔지 않기 위한** 안전장치예요."
  - "Q. 왜 미국 ETF부터 사라고 하나요?" → "A. 한 회사가 아니라 미국 회사 500개를 한 번에 사는 거라, 하나가 망해도 전체는 버티기 때문이에요."
- 효과: ★★★★★ (사용자 요구 "이런 이유로 이렇게 투자하라는 거구나"에 정확히 부합) / 난이도: ★★☆

### F. "떨어지면 어떡하죠?" 섹션 (공포 선제 대응)
- 초보자 이탈 1순위 시나리오(첫 하락장 패닉)를 미리 모의 체험시키는 섹션:
  - 최악의 해 시뮬레이션을 **내 돈 기준 원화**로 보여주기
  - "그때 당신이 할 일: 아무것도 안 하기. 이것도 계획의 일부입니다."
  - 역사적 회복 사실(compliance_rules의 market_history 인용)을 한 줄로
- 효과: ★★★★★ (행동재무학적으로 가장 가치 큰 부분) / 난이도: ★★☆

### G. 실행 카드 (오늘 / 이번 주 / 이번 달)
- 기존 urgent_actions + execution_guide를 시간 축으로 재배열:
  - ✅ **오늘 (5분)**: 파킹통장 개설 버튼 누르기
  - ✅ **이번 주**: 자동이체 30만 원 설정
  - ✅ **이번 달**: 증권사 앱에서 매월 25일 자동매수 설정
- 체크박스 형태로 — "다 읽는 보고서"가 아니라 "하나씩 지우는 할 일 목록"으로 전환
- 효과: ★★★★☆ / 난이도: ★☆☆

### H. HTML 시각화 강화 (Easy HTML 전용)
- **돈 흐름 그림**: 월 여유자금 → 지키는 돈/불리는 돈 → 세부 상품으로 갈라지는 깔때기(텍스트 기반 다이어그램 또는 CSS 막대)
- **신호등 대시보드**: 4지표를 점수 대신 🟢🟡🔴 3색으로만 표시 + 한 줄 코멘트
- **진행률 바**: "비상금 1.7개월 / 목표 3개월" 게이지
- 효과: ★★★★☆ / 난이도: ★★★ (HTML 템플릿 작업 필요)

### I. 대화체 내레이션 (PB와의 가상 상담 형식)
- 리포트 전체를 "친한 PB가 옆에서 말해주는" 1:1 대화체로
- 효과: ★★★☆☆ / 난이도: ★★☆
- ⚠️ 리스크: 길어지면 오히려 가독성 저하. 부분 적용(섹션 도입부 한 줄)만 권장.

### J. 또래 비교 프레임 ("같은 30대 중에서...")
- "당신의 저축률 28%는 또래 평균보다 높은 편이에요" 식 비교
- 효과: ★★★☆☆ (동기 부여에 강력) / 난이도: ★★★★
- ⚠️ 리스크: **실제 통계 데이터 없이 쓰면 환각**. 시스템 철학(수치 창작 금지)과 충돌 → 통계 출처를 compliance_rules에 확보하기 전까지 **보류 권장**.

### K. 용어 미니사전 부록 (Easy 버전 맨 뒤 1단락)
- 본문에선 용어 0 원칙을 지키되, 마지막에 "그래도 궁금하다면: ETF란? ISA란?" 5개만 부록으로
- 효과: ★★★☆☆ / 난이도: ★☆☆

## 4. 추천 조합 — "Easy 리포트 v1" 패키지

> **A(1페이지 요약) + C(숫자 번역) + D(비유 사전) + E(왜요 Q&A) + F(하락 대비) + G(실행 카드)**
> H(시각화)는 v2로, I·J·K는 선택사항으로.

### Easy 리포트 v1 목업 (가상 고객: 35세, 월 여유 100만 원, 73점)

```markdown
# 💌 내 돈 사용설명서 — 한눈에 보기 (2026-06-11)

## ① 지금 내 돈 상태, 한 문장으로
> **"버는 힘은 좋은데, 에어백 통장에 구멍이 있어요."**
> 종합 73점 / 100 (🟡 양호 — 고칠 곳 1~2개. 건강검진으로 치면 '재검 항목 있음' 수준)

## ② 왜 그런가요? (3가지 이유)
1. ✅ 매달 100만 원씩 남습니다 → 아주 잘하고 있는 부분이에요.
2. ⚠️ 그런데 비상금이 생활비 1.7개월치뿐이에요 → 갑자기 큰돈 쓸 일이 생기면,
   투자해 둔 걸 **손해 보면서 팔아야** 할 수 있어요. 이게 지금 가장 큰 구멍입니다.
3. ⚠️ 투자금의 67%가 삼성전자 한 종목에 몰려 있어요 → 달걀이 한 바구니에 있는 상태예요.

## ③ 그래서 이번 달 할 일 (이것만 하면 됩니다)
- [ ] **오늘 (5분)**: 파킹통장 만들기 (증권사·은행 앱에서 'CMA' 또는 '파킹통장' 검색)
- [ ] **이번 주**: 월급날 다음 날 30만 원 자동이체 설정 (비상금 채우기)
- [ ] **이번 달**: 새로 투자할 돈은 삼성전자 추가 매수 대신 미국 S&P500 ETF로
       (앱에서 '정기투자' 메뉴 → 매월 25일 → 49만 원 설정)

## ④ 월 100만 원, 이렇게 나눠요
🛡️ **지키는 돈 30만 원** — 에어백 통장(비상금)과 안전한 채권
📈 **불리는 돈 70만 원** — 그중 49만 원은 '밥'(미국 회사 500개 묶음),
                           21만 원은 '반찬'(관심 종목 소량)

**Q. 왜 불리는 돈이 더 많아요?**
A. 35세는 앞으로 30년 더 벌 수 있는 나이예요. 중간에 떨어져도
   회복할 시간이 충분해서, 지금은 불리는 쪽에 더 두는 게 유리합니다.
   (나이가 들수록 시스템이 자동으로 '지키는 돈'을 늘려갑니다.)

**Q. 왜 미국 회사 500개 묶음(ETF)부터 사라고 해요?**
A. 한 회사 주식은 그 회사가 흔들리면 같이 무너지지만,
   500개 묶음은 몇 개가 망해도 전체는 버티기 때문이에요. 달걀 나눠 담기의 가장 쉬운 방법입니다.

## ⑤ 떨어지면 어떡하죠? (미리 알고 가기)
역사상 최악 수준의 해가 오면, 불리는 돈 70만 원이 **일시적으로 45만 원대로 보일 수 있어요.**
그때 당신이 할 일은 단 하나: **아무것도 안 하기. 이것도 계획의 일부입니다.**
- 에어백 통장(비상금)은 그 하락과 상관없이 그대로예요.
- 과거의 큰 하락(2008년 금융위기 등)도 결국 전부 회복됐어요. 떨어졌을 때 판 사람만 손해를 확정했습니다.

## ⑥ 다음 점검: **2026-09-09** (이날 다시 만나요)

---
📚 그래도 궁금하다면 (미니사전):
ETF = 여러 회사 주식을 한 번에 사는 묶음 상품 / CMA·파킹통장 = 하루만 맡겨도 이자 주는 입출금 통장
※ 자세한 분석 전체는 같은 폴더의 정식 리포트에서 볼 수 있어요.

⚠️ 본 제안은 AI가 생성한 참고용 아이디어이며... (면책 고지 동일 적용)
```

## 5. 시스템 반영 방안 비교 (구현 시점에 선택)

| 방안 | 내용 | 장점 | 단점 | 판단 |
|------|------|------|------|------|
| **A. report-writer 확장** | `templates/master_report_easy.md/.html` 추가 → report-writer가 기존 리포트와 함께 easy 버전도 치환 생성 | 파이프라인 구조 변경 없음. 같은 에이전트가 같은 JSON을 읽으므로 숫자 불일치 위험 최소 | report-writer 프롬프트 비대화. 1회 작업량 증가 | ✅ **권장 (v1)** |
| B. 신규 에이전트 (report-simplifier, Step 6.5) | 완성된 리포트+JSON을 입력으로 '번역 전문' 에이전트 신설 | 역할 분리 명확. '초보자 번역가' 페르소나 최적화 가능 | 에이전트·validate 단계 추가. 숫자가 한 단계 더 전달되며 환각 위험 증가 | v2 후보 |

### 구현 시 반드시 함께 할 것 (정합성 체크리스트)
1. **reviewer V12 확장**: easy 버전의 핵심 수치(점수·금액·비중·날짜)도 소스 JSON과 교차 검증 대상에 포함. "쉽게 번역하다가 숫자가 달라지는" 환각이 가장 큰 리스크.
2. **비유 사전의 단일 진실 공급원화**: `compliance_rules.json`에 `easy_mode_glossary` 블록 추가 — 비유·치환어를 고정해 리포트마다 표현이 달라지는 것 방지.
3. **면책 고지 동일 적용**: easy 버전도 `full_disclaimer` 그대로 포함 (V5 검사 대상 추가).
4. **`validate --agent report`의 플레이스홀더 잔존 검사**에 easy 템플릿 경로 추가.
5. **orchestrator `prepare` 출력 커맨드**에 easy 리포트 경로 안내 추가 (선택).

## 7. 요약 — 한 줄 결론

> **기존 리포트를 고치지 말고, 그 앞에 "1페이지 사용설명서"를 한 장 더 깔아주자.**
> 핵심 공식: **용어 0 + 모든 %는 원화로 + 진단→이유→행동 3박자 + "떨어지면 어떡하죠" 선제 답변 + 체크박스 실행 카드.**
> 구현은 report-writer 확장(방안 A)이 가장 안전하고, 숫자 정합성(V12 확장)만 지키면 환각 리스크 없이 도입 가능.
